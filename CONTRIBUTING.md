# Contributing to Autonoma

Autonoma welcomes contributions. Skills are the highest-leverage contribution because every new skill expands what any Autonoma user can do.

## Contributing a new skill

1. Read `docs/skill-spec.md` to understand the SKILL_SPEC interface
2. Fork this repository
3. Create your skill at `skills/[skill-name]/`
4. Follow the required structure (skill.json, index.js, README.md, tests/, examples/)
5. Ensure all tests pass
6. Open a pull request

Skills are reviewed against the criteria in `docs/skill-review.md`.

## Contributing to the runtime

Runtime improvements, MCP extensions, wallet integrations, and documentation are all welcome. Open an issue before starting significant work so we can align on approach.

## Contributing to documentation

Clear documentation is as valuable as code. Improvements welcome via PR.

## Code style

Follow existing patterns. Keep functions small. Prefer clarity over cleverness. Every skill must be understandable by another developer without asking questions.
